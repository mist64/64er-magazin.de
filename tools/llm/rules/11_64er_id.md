# 11 — Fill `64er.id` with a short lowercase slug per article

**Goal:** replace each article's `<meta name="64er.id" content="XXX<N>">`
placeholder (left by the split step) with a real short lowercase slug
the generator uses for URLs (`/<issue>/<id>.html`), permalinks, and
cross-issue links.

This step has no automatable transformation — the slug is an editorial
decision per article — but the mapping it produces is mechanical to
apply.

## Slug format

- Characters allowed: `a-z`, `0-9`, `-`. Hyphens separate words. No
  spaces, no capitals, no punctuation.
- The project does carry slugs with German umlauts (`dfü`,
  `hexenküche2`) and `_` for older inherited mappings, so accept those
  when matching an existing precedent — but prefer ASCII-hyphen for new
  slugs.
- Soft cap: 24 characters. Longer is allowed when no shorter form is
  recognisable.
- Must be unique within `issues/<YYMM>/`. Across issues, recurring
  rubrics (Aktuelles, Leserforum, Editorial, Fehlerteufelchen,
  Vorschau, CP/M-Ecke, Tips für Einsteiger/Profis/C 16, …) share the
  **same** slug from issue to issue — that's how cross-issue links to
  "the column" stay stable.

## How to pick the slug

1. **Recurring rubric?** Look up the slug used in the previous issue
   for the same column and use it. The recurring slugs include
   `editorial`, `aktuelles`, `leserforum`, `fehlerteufelchen`,
   `vorschau`, `extra`, `cpm`, `dfü`, `pascal`, `assembler`,
   `tips_einsteiger`, `tips_profis`, `tips_c16`, `tips_vizawrite`,
   `superbase`, `anwendung_des_monats`, `listing_des_monats`,
   `knobeleien`, `c128-reisebericht`, `hypra-basic`, `profi-grafik`.
2. **Program / product name?** Use the program's name lowercased and
   hyphenated (e.g. PYRAMIDE → `pyramide`, Vectors → `vectors`,
   Maintext 64 → `maintext`, Seikosha MP-1300 AI → `mp1300`, CPA-80X
   → `cpa80x`).
3. **Topic-leading title?** Use the topical keyword(s) (e.g.
   "Selbstbau Das richtige Kabel zum Monitor" → `monitorkabel`).
4. **Disambiguate** when a natural slug collides with a previous
   article (or with another article in the same issue). Append a
   suffix like `-2`, `-c128`, `-grafik` only after exhausting the
   topic-narrowing options.

The editorial slug should always be `editorial`, regardless of what
the editorial article's title actually reads.

## Procedure

For an issue with the placeholder `XXX0..XXX<N-1>` ids in place:

1. List every article and its title (or topical theme), grouped by
   rubric where applicable.
2. Draft the `{XXX<N> → slug}` mapping. For each article note: the
   article's `<title>`, the chosen slug, and (when not obvious) the
   reason — recurring-rubric match, program name, topic keyword,
   disambiguator.
3. Cross-check the mapping against the previous issue's slugs for
   every recurring rubric — they must match exactly.
4. Apply the mapping to `issues/<YYMM>/*.html`, replacing each file's
   single `<meta name="64er.id" content="XXX<N>">` line with
   `<meta name="64er.id" content="<slug>">`. Leave every other byte
   untouched.

## Verification

```bash
# 1. Every article has a non-empty, non-placeholder id:
for f in issues/<YYMM>/*.html; do
  v=$(grep -oE '<meta name="64er\.id" content="[^"]+"' "$f" | head -1 | sed -E 's/.*content="//;s/"$//')
  [ -z "$v" ] && echo "  empty: $f"
  [[ "$v" =~ ^XXX[0-9]+$ ]] && echo "  placeholder still: $f -> $v"
done

# 2. All ids in the issue are unique:
grep -hoE '<meta name="64er\.id" content="[^"]+"' issues/<YYMM>/*.html | \
  sort | uniq -c | awk '$1 > 1 { print "  dup:", $0 }'

# 3. Charset sanity (lowercase + digits + hyphen + the legacy
#    underscores/umlauts the project tolerates):
grep -hoE '<meta name="64er\.id" content="[^"]+"' issues/<YYMM>/*.html | \
  sed -E 's/.*content="//;s/"$//' | grep -nvE '^[a-z0-9äöüß_-]+$' | \
  head -5

# 4. Cross-issue recurring slugs intact (Aktuelles, Leserforum, …):
for slug in aktuelles leserforum editorial fehlerteufelchen vorschau cpm; do
  prev=$(grep -lE '64er\.id" content="'"$slug"'"' issues/<PREV>/*.html)
  curr=$(grep -lE '64er\.id" content="'"$slug"'"' issues/<YYMM>/*.html)
  [ -n "$prev" ] && [ -z "$curr" ] && echo "  missing recurring slug: $slug"
done
```

All four checks should pass. The cross-issue check matters most when
the issue has the same recurring rubrics — flag missing ones rather
than just silently picking a new slug.

## Evidence-in-report requirement

A previous sub-agent on a different rule claimed verification it never
ran (the `internsiv` OCR regression). To make that failure mode
impossible here, every slug the sub-agent assigns must be backed by
**runnable verifier evidence pasted verbatim into the report**:

- For each recurring-rubric slug, paste the one-line
  `grep -l '64er.id" content="<slug>"' issues/<PREV>/*.html` output
  showing the previous issue used exactly the same slug for the same
  column.
- For each non-recurring slug (program/topic), paste the one-line
  body / `<title>` grep output that justifies the picked slug, e.g.
  ```
  grep -h '<title>' "issues/8607/73 Vectors.html" → "<title>Vectors</title>"  → slug: vectors
  ```
- For every issue-internal uniqueness check, paste the
  `sort | uniq -c | awk '$1>1'` line confirming the chosen slug
  is not already used.

**No verifier output, no claimed slug.** A slug reported without the
previous-issue grep (for recurring rubrics) or the title-derivation
line (for new slugs) is treated as a guess; the orchestrator will
re-dispatch. "Trust me, I matched the rubric" is never acceptable.

## Notes / things to watch

- Don't ship `XXX<N>` as a final id under any circumstances — the
  generator will accept it but every URL will look broken.
- Don't reuse the article's filename as a slug. Filenames are
  sanitised for the filesystem, not for URLs.
- For multi-part articles (e.g. "Pascal-Kurs Teil 4"), keep the
  series slug stable (`pascal`); the part number lives in the title
  and the `64er.toc_title`, not in the id.
- When in doubt between two reasonable slugs, prefer the one that
  matches the article's `<title>` more closely — a future reader
  doing a guess-the-URL is more likely to land on it.
- Slugs are case-sensitive everywhere they appear; "Pyramide" and
  "pyramide" are different URLs.
